# Voxx - Voice Journaling App

## 1. **App Summary**

**Voxx** is a voice journaling app that allows users to record their thoughts and memories using voice entries. Users can create journal entries by recording audio, which is then transcribed and summarized by an AI service. The app presents a clean, simple interface with a list of journal entries, organized from newest to oldest, and each entry includes a play button for the audio and a summary generated by AI. The goal is to make journaling easier, especially for those who prefer speaking over typing.

---

## 2. **User Stories**

### **Story 1: Recording Journal Entries**

As a user,  
I want to be able to record my voice for a journal entry,  
So that I can capture my thoughts quickly without typing.

### **Story 2: Viewing Journal Entries**

As a user,  
I want to view my previous journal entries in a list,  
So that I can easily access and review my thoughts and memories.

### **Story 3: Listening to Journal Entries**

As a user,  
I want to be able to play back the audio of a journal entry,  
So that I can listen to my thoughts as they were originally recorded.

### **Story 4: AI Summary of Journal Entries**

As a user,  
I want my recorded voice entries to be automatically transcribed and summarized by AI,  
So that I can quickly scan through the key points without having to listen to the entire recording.

### **Story 5: Filtering Journal Entries**

As a user,  
I want to filter my journal entries based on date or other attributes,  
So that I can easily find specific entries.

### **Story 6: Editing or Deleting Journal Entries**

As a user,  
I want to be able to edit or delete a journal entry,  
So that I can manage my journal as my thoughts change over time.

## 3. **Data Models**

### **Journal Entry Model**

-   **id**: Unique identifier for each journal entry (auto-increment).
-   **user_id**: The ID of the user who created the journal entry.
-   **audio_url**: The URL where the voice recording is stored (in Firebase).
-   **summary**: AI-generated summary of the recorded voice entry.
-   **created_at**: Timestamp of when the entry was created.
-   **transcription** (optional): Full transcription of the voice recording.

### **User Model** (optional if using Firebase Authentication)

-   **id**: Unique user ID.
-   **email**: User’s email address.
-   **name**: User’s name.

## 4. **Tech Stacks**

### **Frontend (React Native + Expo)**

-   **Expo**: Framework for easy development and deployment of the React Native app.
-   **React Native**: For building the mobile app (cross-platform).
-   **React Navigation**: For handling navigation between screens.
-   **expo-av**: For handling audio recording and playback.
-   **react-native-vector-icons**: For icons (filter icon, play button, etc.).

### **Backend (Supabase + Firebase)**

-   **Supabase**:
    -   Database: For storing metadata about journal entries, summaries, and user information.
    -   Supabase Functions: For handling AI integration (optional for transcription and summarization).
    -   Authentication: For user login/logout (optional if using Firebase Authentication).
-   **Firebase**:
    -   **Firebase Storage**: For storing and retrieving audio recordings.
    -   **Firebase SDK**: For seamless integration with React Native for handling large file uploads.

### **AI Service**

-   **OpenAI API (or other)**: For transcribing and summarizing voice recordings.

---

## 5. **Approach**

### **Step 1: UI Development (React Native + Expo)**

-   Set up the basic structure of the app with a clean, intuitive UI.
-   Implement a list view of journal entries (newest to oldest), a floating action button (FAB) for creating new entries, and playback buttons for each entry.

### **Step 2: Audio Recording & Playback (Expo-AV)**

-   Implement voice recording using the Expo AV library.
-   Allow users to record their thoughts and save the audio file locally before uploading.

### **Step 3: Backend Integration (Firebase & Supabase)**

-   Upload audio files to Firebase Storage and retrieve the audio URL.
-   Use Supabase for storing journal entry metadata (timestamps, audio URLs, AI-generated summaries).
-   Create functions in Supabase to handle CRUD operations for journal entries.

### **Step 4: AI Integration (Supabase Functions or Client-side API Call)**

-   After uploading an audio recording, send the audio file to an AI service (OpenAI) to generate a transcription and summary.
-   Store the AI-generated summary in Supabase and display it in the journal list.

### **Step 5: Testing & Refinement**

-   Test the app on multiple devices and platforms to ensure smooth voice recording, transcription, and playback.
-   Optimize performance, particularly in handling audio files and AI requests.
-   Ensure proper error handling for network issues, API failures, and large file uploads.

### **Step 6: Deployment**

-   Deploy the app to the iOS and Android app stores using Expo’s managed workflow.
